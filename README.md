# Accelerometer app

### Реализованные пункты задания:
Уровень сложности 2.

Все сервисы полностью на Linux, в узле а сигнал акселерометра эмулируется синусоидой.
## Архитектура:
Микросервисы, общающиеся черзе gRPC+Protobuf.

Настройка частоты сенсора выведена в конфиг файл.

В узле А генерируется сигнал через синусоиды и увеличивающийся счетчик:

x = sin(counter)*2; 
y = sin(counter)*9.8; 
z = sin(counter*0.5)*0.5; 

counter += 0.1;

Узел А отправляет данные на сервер. Сервер их фильтрует, отправляет на узел Б, который считает ускорение и отправляет данные обратно.

Общение реализовано на базе grpc Callback API асинхронно. 

Каждый микросервис завернут в класс-оркестратор, отвечающий за корректное переподключение.
## Инструкции по сборке
Используется система сборки CMake с модулем FetchContent. 

Зависимости: 
```
CMake >= 3.16
C++ >= 17
gRPC v1.62.0
Protobuf v26.0
```

Система автоматически проверяет наличие grpc и protobuf, при необходимости их устанавливает.

Сборка:

```CMake
cmake -B build -S .
cmake --build build
```

Итоговые бинарники в папках 

```
build/server/server
build/node_a/node_a
build/node_b/node_b
```

## Запуск
Настройка частоты работы сенсора в конфиг файле в корневой папке:
```config.txt
sensor_frequency_hz=50
```
При запуске первым аргументом командной строки передается порт(для сервера) и адрес(для клиента)
Запуск из корня проекта:
```
# Терминал 1: Запуск сервера на порту 9090
./build/server/server 9090

# Терминал 2: Запуск Узла А (Сенсора) с подключением к серверу
./build/node_a/node_a 127.0.0.1:9090

# Терминал 3: Запуск Узла Б (Обработчика)
./build/node_b/node_b 127.0.0.1:9090
```
